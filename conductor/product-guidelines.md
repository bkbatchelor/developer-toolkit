# Product Guidelines

## Prose Style: Educational & Friendly
Our documentation should be engaging and easy to follow. We aim to explain technical concepts clearly while maintaining a friendly and approachable tone. Avoid overly dense jargon and prioritize readability.

## Branding: Modern & Minimalist
The project's branding focuses on a clean, simple, and functional aesthetic. We use standard Markdown formatting with consistent headers, lists, and code blocks. The goal is to let the content and code shine without unnecessary visual distractions.

## UX Principles: Documentation-Heavy
We believe that clear documentation is as important as the code itself. Each module (`persistence-sql/`, `persistence-nosql/`) will feature a comprehensive README explaining its specific role, design decisions, and implementation details.

## Tone & Voice: Mentoring
Our voice is that of a mentor—guiding the user through the complexities of SQL vs. NoSQL architectures. We adopt a helpful and authoritative yet supportive tone, anticipating questions and providing clear, actionable insights to help developers make informed decisions.
