# Java Code Style Guide (Google Style)

## 1. Source File Organization
MUST adhere to the `style-guide-adherence` skill.

## 2. Formatting
- **Indentation:** MUST adhere to the `style-guide-adherence` skill.
- **Line Length:** MUST adhere to the `style-guide-adherence` skill.
- **Braces:** MUST adhere to the `style-guide-adherence` skill.

## 3. Naming Conventions
- **Class Names:** MUST adhere to the `style-guide-adherence` skill.
- **Method Names:** MUST adhere to the `style-guide-adherence` skill.
- **Variables & Parameters:** MUST adhere to the `style-guide-adherence` skill.
- **Constants:** MUST adhere to the `style-guide-adherence` skill.
- **Package Names:** MUST adhere to the `style-guide-adherence` skill.

## 4. Programming Practices
- **TDD Mandate:** MUST adhere to the `tdd` skill.
- **Overrides:** MUST adhere to the `style-guide-adherence` skill.
- **Exceptions:** MUST adhere to the `style-guide-adherence` skill.
- **Static Members:** MUST adhere to the `style-guide-adherence` skill.
- **Lombok:** Use Lombok annotations responsibly to reduce boilerplate (e.g., `@Data`, `@Getter`, `@Setter`) if not in conflict with the `style-guide-adherence` skill.

## 5. Documentation
- **Javadoc:** MUST adhere to the `style-guide-adherence` skill.
- **Block Tags:** MUST adhere to the `style-guide-adherence` skill.
